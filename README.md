# Restaurant-Gemini

The system is client-server architecture for a **Restaurant Menu & Ordering System**. The system is designed to replace paper-based menus and handle the complete lifecycle of a customer's order.

Based on the foundational MVP (Minimum Viable Product) requirements identified for the hypothetical first sprint, here are the specific requirements included from the Software Requirements Specification:

- **Infrastructure & General Setup**

- **Security & Authentication**

- **Basic Customer UI**




## Deployment Instructions

1. Ensure **Docker** and **Docker Compose** are installed on your machine.
2. Open a terminal and navigate to the `rmos-sprint1` root directory you created.
3. Run the following command to build and start the containers:
```bash
docker-compose up --build

```

4. Open the prototype UI:

   * Frontend: `http://localhost`
